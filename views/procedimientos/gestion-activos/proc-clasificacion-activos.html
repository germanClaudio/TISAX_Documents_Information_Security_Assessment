Procedimiento para Clasificación y Gestión de Activos de Información

Código: P132-IT-1
Última revisión: 25/8/2025
Rev. N°: 0
Cumple: TISAX AL2, ISO 27001:2022

1 OBJETIVO

Establecer un esquema de clasificación coherente y medidas de protección alineadas con los requisitos de Confidencialidad, Integridad y Disponibilidad (CID), para cumplir con los requisitos de seguridad de TISAX, ISO/IEC 27001:2022 y de clientes automotrices.

2 ESQUEMA DE CLASIFICACIÓN DE ACTIVOS

Basado en criticidad y requisitos VDA/ISO 27001:

Nivel Definición Ejemplos en Prodismo Etiqueta
Crítico Exposición causa daño financiero/legal grave (>USD 500k) o pérdida de IP.
  • Diseños 3D (CATIA/NX)
  • BD clientes (SIP/SIP4.0)
 🔴 Rojo
Alto Impacto operacional severo (interrupción >24h) o violación de regulaciones.
  • Windows Server
  • Correos ejecutivos
 🟡 Amarillo
Medio Afecta procesos no críticos (interrupción <24h).
  • Documentos en SharePoint
  • MS Teams
 🔵 Azul
Bajo Impacto mínimo o nulo.
  • Manuales internos obsoletos
 ⚪ Blanco

Criterios CID:

Confidencialidad:

  • 🔴 Solo personal autorizado (ej.: ingenieros para CATIA).
  • 🟡 Equipos específicos (ej.: financiero para SIP).

Integridad:

  • 🔴 Validación con firma digital (ej.: diseños CATIA).
  • 🟡 Revisiones semanales (ej.: logs de acceso a servidores).

Disponibilidad:

  • 🔴 SLA 99.99% (ej.: AWS para CATIA).
  • 🟡 SLA 99.9% (ej.: Office 365).

3 PROCESO DE EVALUACIÓN Y CLASIFICACIÓN

Pasos:

  1. Identificación:
    • Listar activos en inventario (ej.: INF-012: CATIA, SOP-005: Windows Server).
  2. Evaluación:
    • Usar tabla de criterios CID (ver arriba).
  3. Asignación:
    • Etiquetar activos según esquema (🔴/🟡/🔵/⚪).
  4. Documentación:
    • Actualizar Inventario de Activos con:
      • Nivel de clasificación.
      • Responsable asignado.
      • Controles aplicados (ej.: "Cifrado AES-256 para 🔴").

Herramientas:

  • Plantilla Excel o software GRC (Snipe-IT).

4 GESTIÓN DE ACTIVOS DE SOPORTE

Especificaciones por Tipo:

Activo de Soporte Requisitos para 🔴 (Crítico) Requisitos para 🟡 (Alto)
Dispositivos
  • Cifrado Full Disk (BitLocker).
  • Inventario con GPS.
  • Cifrado opcional.
  • Registro básico.
Almacenamiento
  • NAS con redundancia + copia offline.
  • Acceso biométrico.
  • Backup diario en cloud.
Transporte
  • USB cifrados + log de movimientos.
  • USB con contraseña.
Eliminación
  • Borrado seguro (NIST 800-88).
  • Certificado de destrucción.
  • Formateo estándar.

Ejemplo para CATIA (🔴):

  • Almacenamiento: Server con cifrado + watermarking en archivos.
  • Transporte: Archivos compartidos vía Siemens Teamcenter (no email).
  • Eliminación: Borrado con "Blancco" + certificado.

5 IMPLEMENTACIÓN DE CONTROLES

Por Nivel de Clasificación:

Control 🔴 (Crítico) 🟡 (Alto)
Accesos MFA + IP whitelisting MFA obligatorio
Cifrado AES-256 + HSMs AES-256
Backups 3-2-1 rule (3 copias, 2 medios, 1 offsite) Backup diario
Registro de Accesos Logs con SIEM + alertas Logs mensuales

6 MONITOREO Y AUDITORÍA

Revisiones Periódicas:

  • 🔴 Trimestral (por IT Manager).
  • 🟡 Semestral (por Equipo IT).

Auditorías Internas:

  • Verificar etiquetado correcto (ej.: ¿CATIA está como 🔴?).
  • Validar controles (ej.: ¿MFA activo para MS Teams?).

Evidencias para TISAX:

  1. Inventario de activos con clasificación CID.
  2. Registros de revisiones y auditorías.
  3. Certificados de destrucción/eliminación.

Ejemplo de Documentación

Inventario de Activos (Extracto):

ID Nombre Tipo Clasificación Responsable Controles
INF-012 Diseños CATIA Información 🔴 Crítico [Nombre] Cifrado AES-256, MFA, watermarking.
SOP-005 Windows Server Soporte 🟡 Alto [Nombre] Parches automáticos, logs SIEM.

Recomendaciones Clave

  • Automatización: Usar herramientas como Microsoft Purview o Snipe-IT para etiquetado automático.
  • Capacitación: Entrenar a empleados en manejo de 🔴 (ej.: no enviar archivos CATIA por email).
  • Integración con Sistema de Gestión de la Seguridad de la Información (SGSI): Vincular clasificación al "F121-IT-2 Informe Declaración de Aplicabilidad (DoA)".

Instructivo de uso de Snipe-IT

  1. Acceso: Ingresar a la plataforma Snipe-IT con las credenciales proporcionadas.
  2. Clasificación de activos:
    • Seleccionar el activo a clasificar en la lista de "Assets".
    • Hacer clic en "Edit" para modificar sus propiedades.
    • En el campo "Classification", seleccionar el nivel correspondiente (Crítico, Alto, Medio, Bajo).
    • Guardar los cambios.
  3. Etiquetado automático:
    • Configurar reglas de clasificación automática basadas en tipos de archivo o ubicación.
    • Establecer alertas para activos sin clasificar.
  4. Reportes de clasificación: Generar reportes mensuales de activos por nivel de clasificación desde la sección "Reports".

Para soporte técnico de Snipe-IT, contactar al administrador del sistema.
