Procedimiento de Gestión de Servicios de IT Externos

Código: P133-IT-1
Última revisión: 25/8/2025
Rev. N°: 0
Cumple: TISAX AL2, ISO 27001:2022

1 OBJETIVO

Asegurar que servicios cloud, SaaS, o proveedores externos cumplan con los requisitos de seguridad de la información, especialmente para datos críticos (CATIA, SIP/SIP4.0) y clientes automotrices, cumpliendo con los requisitos TISAX, ISO/IEC 27001:2022 y de clientes automotrices.

2 EVALUACIÓN DE RIESGOS PARA SERVICIOS EXTERNOS

Checklist Inicial:

Criterio Evaluación Ejemplo
Tipo de Datos Procesados ¿Maneja datos 🔴 Críticos (CATIA) o 🟡 Altos (SIP)? Google Drive almacena diseños 3D.
Ubicación del Servidor ¿Cumple GDPR si almacena datos de la UE? AWS región Frankfurt (UE).
Certificaciones ¿Tiene ISO 27001, TISAX, o SOC 2? Microsoft 365 cumple ISO 27001.
Controles de Seguridad ¿Ofrece MFA, cifrado, y logs de acceso? TeamViewer registra IPs de acceso.

Herramienta:

Matriz de riesgo con puntuación (ej.: 1-5) por criterio.

3 REQUISITOS LEGALES Y CONTRACTUALES

Cláusulas Obligatorias en Contratos:

  • Protección de Datos:
    "El Proveedor garantizará cifrado AES-256 para datos en tránsito/reposo, cumplimiento con VDA ISA para diseños CATIA."
  • Acceso y Auditoría:
    "El Proveedor permitirá auditorías anuales por parte de [Organización] o clientes (ej.: BMW GS.11)."
  • Respuesta a Incidentes:
    "Notificación de brechas en <24h, con remediación en 72h."

Ejemplo de Requisitos por Cliente:

Cliente Exigencias Clave
VW Prohibido usar Dropbox/WeTransfer (solo plataformas aprobadas).
BMW Watermarking en archivos CAD compartidos.
Mercedes Retención de logs por 6 meses mínimo.

4 ARMONIZACIÓN CON NECESIDADES DE PROTECCIÓN

Matriz de Alineación:

Servicio Externo Activos Asociados Controles Requeridos Estado
AWS S3 CATIA/NX (🔴) Cifrado + MFA + watermarking. ✅ Implementado
TeamViewer Soporte Remoto (🟡) Logs de acceso + IP whitelisting. ⚠️ Pendiente MFA
Zoom Reuniones (🔵) Cifrado E2E + grabaciones locales. ❌ No aprobado

Acciones para No Aprobados:

  • Zoom: Reemplazado por Microsoft Teams (cumple TISAX).

5 PROCEDIMIENTO DE ADQUISICIÓN

Flujo de Aprobación:

  1. Solicitud: Equipo de proyecto envía formulario con:
    • Datos involucrados (ej.: "Subir CATIA a AWS").
    • Clasificación CID (🔴/🟡/🔵).
  2. Revisión por IT Manager:
    • Verificar checklist de riesgos.
    • Validar contrato con cláusulas de seguridad.
  3. Implementación:
    • Configurar controles (ej.: MFA en AWS).
    • Capacitar usuarios (ej.: no compartir links públicos en SharePoint).

6 MONITOREO Y AUDITORÍA

Herramientas:

  • Cloud Access Security Broker (CASB): Monitorear uso de servicios no autorizados (ej.: empleados usando Google Drive).
  • Informes Trimestrales:
    • % de servicios externos cumpliendo TISAX.
    • Incidentes relacionados (ej.: acceso no autorizado a AWS).

Ejemplo de Rechazo de Servicio

Servicio: Zoom

Motivo:

  • No cumple cifrado E2E para reuniones con diseños CATIA (requerido por BMW).

Alternativa Aprobada: Microsoft Teams con:

  • Grabaciones locales.
  • MFA obligatorio.

Recomendaciones Clave

  • Lista Blanca/Aprobada: Publicar en intranet servicios externos autorizados (ej.: AWS, MS Teams).
  • Penalizaciones: Incluir en contratos multas por incumplimiento (ej.: 3% del pago).
  • Training: Capacitar en riesgos de "Shadow IT" (ej.: no usar Dropbox para CATIA).
