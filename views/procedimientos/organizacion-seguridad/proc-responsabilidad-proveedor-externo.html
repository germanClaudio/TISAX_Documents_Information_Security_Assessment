1 OBJETIVO

Establecer acuerdos claros y verificables con proveedores externos (ej.: iCloud, soporte técnico) para cumplir con los requisitos de seguridad de TISAX, ISO/IEC 27001:2022 y de clientes automotrices.

  • Diseños 3D

3 REQUISITOS DE SEGURIDAD POR PROVEEDOR

Matriz de Controles Obligatorios:

Proveedor Requisito TISAX Responsable (Org/Proveedor) Mecanismo de Verificación
Prodismo Cifrado AES-256 para CATIA Organización (IT Interno) Certificación TISAX - ISO27000
TeamViewer Registro de logs de acceso Organización (IT Interno) Reporte mensual de actividad
Microsoft MFA para Office 365 Compartido (MS + IT Interno) Auditoría Azure AD

Ejemplos de Requisitos Específicos por Cliente:

  • VW: Prohibido usar Anydesk (solo TeamViewer con aprobación).
  • BMW: Watermarking en archivos compartidos vía SharePoint.
  • Mercedes: Retención de logs de acceso por 6 meses mínimo.

5 MECANISMOS DE IMPLEMENTACIÓN

Herramientas y Procesos:

Contratos y SLAs:

  • Cláusula de cumplimiento TISAX y normativas de clientes (ej.: "BMW GS.11").
  • Penalizaciones por incumplimiento (ej.: 5% del pago por falta de cifrado).

Acceso a Evidencias:

  • Dashboard compartido.
  • Reuniones trimestrales de revisión (minutas documentadas).

6 MONITOREO Y CUMPLIMIENTO

Checklist de Verificación:

Ítem Frecuencia Responsable
Revisión de logs de acceso (TeamViewer) Semanal Administrador IT
Auditoría de cifrado (Interno) Semestral IT Manager
Validación de MFA (Office 365) Mensual Soporte IT

Documentación Requerida para TISAX AL2

  1. Inventario de proveedores con clasificación de riesgos.
  2. Acuerdos firmados (SLAs, NDAs) que incluyan:
    • Requisitos de seguridad específicos (ej.: " Microsoft debe cumplir ISO 27017").
    • Protocolos de respuesta a incidentes (ej.: notificar en <24h).
  3. Registros de auditorías (ej.: informe de cumplimiento de Microsoft).

Recomendaciones Clave

  • Para VW/BMW/Mercedes:
    • Exigir certificaciones específicas (ej.: TISAX para proveedores de CAD).
    • Incluir cláusulas de auditoría sorpresa en contratos.
  • Automatizar: Usar Azure Policy para verificación en tiempo real.
