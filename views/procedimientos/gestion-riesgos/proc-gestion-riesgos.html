Rev. N°: 0
Cumple: TISAX AL2, ISO 27001:2022

1 OBJETIVO

Implementar un proceso ágil y efectivo para identificar, evaluar y mitigar riesgos, enfocado en activos críticos (ej.: CATIA/NX, SIP) en cumplimiento de ISO/IEC 27001:2022, así como los requisitos del marco TISAX y cumplimiento de requisitos de clientes automotrices.

2 EVALUACIÓN DE RIESGOS

Metodología:

Basada en ISO 27005 y VDA ISA, adaptada a entornos on-premise.

Pasos:

  1. Identificación de Activos y Amenazas:
    • Activos Críticos: CATIA/NX (🔴), SIP (🔴), Windows Server (🟡).
    • Amenazas Comunes:
      • Ransomware en servidores de diseño.
      • Acceso no autorizado a SIP por phishing.
  2. Evaluación de Probabilidad e Impacto:
    • Escala 1-5 (1=Bajo, 5=Crítico).
    • Ejemplo:
    Riesgo Probabilidad Impacto Nivel de Riesgo
    Ataque ransomware a CATIA 4 5 Alto (20)
    Fuga de datos de SIP por phishing 3 4 Medio (12)
  3. Documentación:
    • Registro de Riesgos (Plantilla):
    ID Riesgo Activo Afectado Clasificación Responsable Controles Existentes Acciones de Mitigación
    R1-IT-01 Ransomware en CATIA Servidor On-Premise Alto (20) Admin TI Backup diario Segmentar red + EDR

3 GESTIÓN DE RIESGOS

Matriz de Tratamiento:

Riesgo Estrategia Acciones Específicas Plazo
Ransomware en CATIA Mitigar
  • Implementar segmentación VLAN para ingeniería.
  • Desplegar EDR (CrowdStrike).
 30 días
Phishing a SIP Transferir
  • Contratar seguro cibernético (cobertura USD 1M).
 60 días

Responsables:

  • Dueño del Riesgo: Asignado según área (ej.: Admin TI para riesgos de infraestructura).
  • IT Manager: Supervisa la implementación de controles.

4 REVISIONES PERIÓDICAS

Frecuencia:

  • Trimestral: Riesgos 🔴 Críticos (ej.: CATIA).
  • Semestral: Riesgos 🟡 Altos/🔵 Medios.
  • Ad-hoc: Tras incidentes o cambios (ej.: nuevo cliente BMW con requisitos adicionales).

Checklist de Revisión:

  • ¿Los controles mitigaron el riesgo? (Verificar con métricas).
  • ¿Cambió la probabilidad/impacto? (Actualizar matriz).
  • ¿Se asignaron nuevos responsables?

5 COMUNICACIÓN Y MONITOREO

Herramientas:

  • Dashboard de Riesgos: Power BI o Excel con:
    • Nivel de riesgo actualizado.
    • Estado de acciones de mitigación.
  • Alertas Automáticas: Integrar SIEM (ej.: detección de intentos de acceso fuera de horario).

Ejemplo Práctico

Caso: Nuevo requisito de BMW para watermarking en CATIA

  1. Identificar Riesgo: "Fuga de diseños sin watermarking" → Impacto: 5 (Crítico).
  2. Mitigación:
    • Instalar Siemens NX con módulo de watermarking.
    • Capacitar a ingenieros/diseñadores.
  3. Responsable: Responsable de Datos.

Recomendaciones

  • Automatización: Usar Microsoft Defender for Endpoint para detección de ransomware.
  • Integración con TISAX: Vincular riesgos a la Declaración de Aplicabilidad (DoA).
  • Ejemplo de Clientes Automotrices:
    • VW: Enfocarse en protección física de servidores on-premise.
    • BMW: Priorizar watermarking y logs detallados.
