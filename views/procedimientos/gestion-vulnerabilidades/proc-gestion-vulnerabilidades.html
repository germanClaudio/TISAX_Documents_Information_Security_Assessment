Procedimiento de Gestión de Vulnerabilidades

Código: P525-IT-1
Última revisión: 25/8/2025
Rev. N°: 0
Cumple: TISAX AL2, ISO 27001:2022

1 OBJETIVO

Establecer un proceso sistemático para identificar, evaluar y corregir vulnerabilidades en sistemas IT (ERP SIP y SIP 4.0, aplicaciones internas, redes), minimizando riesgos de ciberseguridad, en cumplimiento con ISO/IEC 27001:2022, TISAX y clientes automotrices.

2 IDENTIFICACIÓN DE VULNERABILIDADES

A. Alcance:

Sistemas críticos:

  • ERP SIP y SIP 4.0 (versiones, parches).
  • Aplicación interna (dependencias de código).
  • Infraestructura (servidores, firewalls).

B. Proceso:

  1. Fuentes de información:
    • Bases de datos públicas (NVD, CVE).
    • Alertas de proveedores (ej.: notificaciones de parches para SIP 4.0).
  2. Herramientas:
    • Escaneos automáticos mensuales con OpenVAS o Nessus.
    • Revisión manual de dependencias en código (ej.: npm audit para app interna).

C. Documentos de Referencia:

  • F525-IT-1 Inventario de Sistemas (Listado de activos a escanear).
  • F525-IT-2 Reporte Escaneo de Vulnerabilidades (Resultados de vulnerabilidades detectadas).

3 EVALUACIÓN Y PRIORIZACIÓN

A. Alcance:

  • Clasificar vulnerabilidades según impacto y urgencia.

B. Proceso:

  1. Criticidad:
    • Usar CVSS v3.1 (ej.: puntaje ≥7.0 = Crítico).
  2. Contexto empresarial:
    • ¿Afecta a datos financieros (SIP 4.0)? ¿Es explotable remotamente?
  3. Matriz de decisión:
    • Críticas: Parchear en ≤72 horas.
    • Medias: Planificar en ≤2 semanas.

C. Documentos de Referencia:

  • M525-IT-1 Matriz de Priorización CVSS (Matriz CVSS vs. sistemas afectados).

4 MITIGACIÓN/CORRECCIÓN

A. Alcance:

  • Resolución de vulnerabilidades priorizadas.

B. Proceso:

  1. Parches:
    • Aplicar updates de proveedores (ej.: parches SAP para SIP 4.0).
  2. Workarounds:
    • Si no hay parche: Aislar sistema o deshabilitar funciones vulnerables.
  3. Validación:
    • Re-escanear tras correcciones.

C. Documentos de Referencia:

  • P525-IT-4 Procedimiento de Aplicación de Parches (Flujo de actualizaciones).

5 MONITORIZACIÓN CONTINUA

A. Alcance:

  • Detección proactiva de nuevas vulnerabilidades.

B. Proceso:

  1. Suscripciones:
    • Alertas RSS de CVE para tecnologías usadas (ej.: Windows Server, SAP).
  2. Auditorías trimestrales:
    • Revisión manual de configuraciones (ej.: puertos abiertos en firewall).

C. Documentos de Referencia:

  • MC525-IT-5 Monitorización de Vulnerabilidades (Frecuencia y responsables).

Recomendaciones para Implementación Eficiente

  1. Herramientas de bajo costo:
    • OpenVAS (gratuito) para escaneos automatizados.
    • GitHub Dependabot para alertas en código de la app interna.
  2. Proveedores:
    • Exigir SLA de parches para SIP 4.0 en contratos.
