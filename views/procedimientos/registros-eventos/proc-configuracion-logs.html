Procedimiento de Configuración de Logs

Código: P524-IT-2
Última revisión: 25/8/2025
Rev. N°: 0
Cumple: TISAX AL2, ISO 27001:2022

1 OBJETIVO

Establecer el proceso sistemático para la implementación, configuración y mantenimiento de la generación de registros (logs) en todos los sistemas de información, asegurando el cumplimiento de los requisitos definidos en la Política de Retención de Logs (MC524-IT-2) y la norma ISO/IEC 27001:2022, así como con los requisitos de evidencia del marco TISAX y clientes automotrices.

2 ALCANCE

Este procedimiento aplica a:

  • Todos los sistemas nuevos que se incorporen a la infraestructura de IT
  • Sistemas existentes que requieran actualización de configuración de logs
  • Plataformas: Servidores Windows/Linux, ERP SIP 4.0, aplicaciones internas, dispositivos de red, servicios en la nube
  • Personal: Administradores de sistemas, desarrolladores, equipo de seguridad

3 DEFINICIONES

  • Syslog: Protocolo estándar para el envío de mensajes de log
  • WEF (Windows Event Forwarding): Tecnología para centralizar logs de Windows
  • Agente de Log: Software instalado en sistemas para enviar logs al SIEM
  • Nivel de Log: Grado de detalle de la información registrada (DEBUG, INFO, WARN, ERROR)

4 REQUISITOS MÍNIMOS DE CONFIGURACIÓN

4.1. Configuración General para Todos los Sistemas

  • Sincronización horaria: Configurar NTP con servidor corporativo
  • Formato de timestamp: ISO 8601 (YYYY-MM-DD HH:MM:SS TIMEZONE)
  • Codificación de caracteres: UTF-8
  • Rotación de logs: Implementar para evitar llenado de discos

4.2. Niveles de Log Requeridos

Tipo de Sistema Nivel Mínimo Eventos Críticos a Registrar
Servidores Críticos INFO Autenticación, cambios de configuración, errores del sistema
ERP SIP 4.0 INFO Transacciones financieras, acceso a datos sensibles, cambios de permisos
Aplicaciones Internas WARN Excepciones de seguridad, accesos no autorizados, errores de negocio
Dispositivos de Red INFO Cambios de configuración, intentos de acceso, eventos de seguridad

5 PROCEDIMIENTO DE CONFIGURACIÓN

5.1. Fase 1: Planificación y Diseño

Actividades:

  1. Identificar sistemas sujetos a configuración de logging
  2. Definir eventos específicos a registrar según criticidad del sistema
  3. Establecer formatos de log estándar
  4. Diseñar arquitectura de envío al SIEM centralizado

Entregables:

  • Matriz de eventos a registrar por sistema
  • Diagrama de arquitectura de logging
  • Plan de implementación

5.2. Fase 2: Implementación Técnica

5.2.1. Servidores Windows

                                        # Configuración mediante Group Policy
                                        - Habilitar auditoría detallada
                                        - Configurar WEF para envío a SIEM
                                        - Establecer tamaño máximo y rotación de logs

5.2.2. ERP SIP 4.0

  • Habilitar logging de transacciones críticas
  • Configurar auditoría de acceso a datos sensibles
  • Establecer envío de logs via agente al SIEM

5.2.3. Dispositivos de Red

                                        # Configuración estándar
                                        logging host [IP-SIEM]
                                        logging trap informational
                                        logging source-interface [interface]

5.3. Fase 3: Verificación y Validación

Actividades:

  1. Generar eventos de prueba para verificar registro
  2. Validar recepción en SIEM centralizado
  3. Verificar formato y contenido de logs
  4. Documentar configuración implementada

Criterios de Aceptación:

  • Logs recibidos en SIEM dentro de los 5 minutos del evento
  • Formato de timestamp correcto
  • Información mínima requerida presente

6 PLANTILLAS DE CONFIGURACIÓN ESTÁNDAR

6.1. Plantilla para Servidores Windows

                                    <!-- Configuración GPO para Logging -->
                                    <Configuration>
                                        <AuditPolicy>
                                            <AuditLogon enabled="true"/>
                                            <AuditObjectAccess enabled="true"/>
                                            <AuditPolicyChange enabled="true"/>
                                        </AuditPolicy>
                                        <EventLog>
                                            <MaxSize>2048MB</MaxSize>
                                            <Retention>90</Retention>
                                        </EventLog>
                                    </Configuration>

6.2. Plantilla para Aplicaciones Web

                                    {
                                        "logging": {
                                            "level": "INFO",
                                            "format": "TIMESTAMP LEVEL [THREAD] CLASS - MESSAGE",
                                            "appenders": [
                                                {
                                                    "type": "file",
                                                    "file": "/var/log/applications/app.log",
                                                    "maxSize": "100MB",
                                                    "maxFiles": 10
                                                }
                                            ]
                                        }
                                    }

7 MONITOREO Y MANTENIMIENTO

7.1. Verificaciones Periódicas

Actividad Frecuencia Responsable
Verificación de logs activos Diaria Equipo de SOC
Revisión de configuración Mensual Administradores de sistemas
Actualización de plantillas Trimestral RSI

7.2. Indicadores de Performance

  • Tasa de generación de logs: Volumen normal por sistema
  • Tiempo de entrega: Máximo 5 minutos desde evento a SIEM
  • Cobertura de sistemas: Porcentaje de sistemas con logging configurado

8 GESTIÓN DE INCIDENTES DE LOGGING

8.1. Detección de Problemas

  • Falta de logs: Alertar cuando un sistema deje de enviar logs
  • Formato incorrecto: Detectar logs que no cumplan estándares
  • Volumen anómalo: Alertar por picos o caídas significativas

8.2. Procedimiento de Corrección

  1. Detectar anomalía en generación/recepción de logs
  2. Investigar causa raíz (configuración, red, sistema)
  3. Aplicar corrección según procedimientos establecidos
  4. Verificar restauración del servicio de logging
  5. Documentar incidente y solución aplicada

10 REVISIÓN Y AUDITORÍA

10.1. Revisiones Periódicas

  • Este procedimiento será revisado anualmente por el RSI
  • Se actualizarán las plantillas según cambios tecnológicos
  • Se incorporarán lecciones aprendidas de incidentes

10.2. Auditorías de Cumplimiento

  • Verificación técnica mensual de configuraciones activas
  • Auditoría interna semestral del proceso completo
  • Inclusión en evaluaciones TISAX y auditorías ISO 27001

10.3. Métricas de Control

  • Porcentaje de sistemas con logging configurado correctamente
  • Tiempo promedio de detección de fallos en logging
  • Número de incidentes relacionados con falta de logs
